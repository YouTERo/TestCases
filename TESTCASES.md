# Позитивные тест кейсы
## TC-1. Создание объявления с валидными данными

**Метод**: Post

**Эндпоинт**: /api/1/item

**Описание**: Создание объявления с валидными данными (sellerId: 452612, name: ""iPhone 15"", price: 100000)

**Ожидаемый результат**: HTTP 200, JSON содержит id (UUID), createdAt, данные совпадают с отправленными.

## TC-2. Получение существующего объявления по корректному id.

**Метод**: Get

**Эндпоинт**: /api/1/item/{id}

**Описание**: Получение существующего объявления по корректному id = 452612.

**Ожидаемый результат**: HTTP 200, тело ответа содержит все поля объявления, данные корректны.

## TC-3. Получение списка всех объявлений для продавца по id.

**Метод**: Get

**Эндпоинт**: /api/1/{sellerID}/item

**Описание**: Получение списка всех объявлений для продавца (например, 452612).

**Ожидаемый результат**: HTTP 200, возвращается массив объектов, у всех sellerId равен запрошенному.

## TC-4. Получение статистики по существующему item_id.
**Метод**: Get

**Эндпоинт**: /api/1/statistic/{id}

**Описание**: Получение статистики по существующему item_id.

**Ожидаемый результат**: HTTP 200, ответ содержит поля likes, viewCount, contacts.

## TC-5. Создание объявления с граничными значениями
**Метод**: Post

**Эндпоинт**: /api/1/item

**Описание**: Создание объявления с минимально допустимыми значениями (price: 0, название из 1 символа).

**Ожидаемый результат**: HTTP 200, объявление успешно создано.
# Негативные тест кейсы
## TC-6. Создание объявления с пропуском обязательного поля
**Метод**: Post

**Эндпоинт**: /api/1/item

**Описание**: Пропуск обязательного поля (например, name).

**Ожидаемый результат**: HTTP 400 Bad Request, понятное сообщение об ошибке.
## TC-7. Создание объявления с некорректным типом данных
**Метод**: Post

**Эндпоинт**: /api/1/item

**Описание**: Передача некорректного типа данных (например, price: ""free"" — строка вместо числа).

**Ожидаемый результат**: HTTP 400 Bad Request.
## TC-8. Запрос несуществующего id объявления.
**Метод**: Get

**Эндпоинт**: /api/1/item/{id}

**Описание**: Запрос несуществующего id (валидный UUID, но нет в базе)

**Ожидаемый результат**: HTTP 404 Not Found
## TC-9. Запрос с некорректным форматом id объявления
**Метод**: Get

**Эндпоинт**: /api/1/item/{id}

**Описание**: Запрос с некорректным форматом id (не UUID, например 123-abc)

**Ожидаемый результат**: HTTP 400 Bad Request
## TC-10. Запрос объявлений несуществующего продавца.
**Метод**: Get

**Эндпоинт**: /api/1/{sellerID}/item

**Описание**: Запрос с некорректным форматом id (не UUID, например 123-abc)

**Ожидаемый результат**: HTTP 200, возвращается пустой список []
## TC-11. Запрос некорректного sellerID.
**Метод**: Get

**Эндпоинт**: /api/1/{sellerID}/item

**Описание**: Передача в sellerID строки вместо числа.

**Ожидаемый результат**: HTTP 400 Bad Request
# Корнер-кейсы
## TC-12. Создание двух объявлений подряд с абсолютно идентичным телом
**Метод**: Post

**Описание**: Создание двух объявлений подряд с абсолютно идентичным телом (name, price, sellerId)

**Ожидаемый результат**: Создаются два разных объявления с уникальными id.

## TC-13. Выполнение 5 последовательных GET запросов к одному и тому же id. (Идемпотентность)
**Метод**: GET

**Описание**: Выполнение 5 последовательных GET запросов к одному и тому же id.

**Ожидаемый результат**: Состояние системы не меняется, ответ всегда идентичен.

## TC-14. Создание объявления с экстремально большой ценой
**Метод**: Post

**Описание**: Создание объявления с экстремально большой ценой 2^63 - 1

**Ожидаемый результат**: Если поле типа Long — успех, если Overflow — 400 ошибка.

## TC-15. Удаление объявления и попытка получить его снова через GET.
**Метод**: Delete

**Описание**: Удаление объявления и попытка получить его снова через GET.

**Ожидаемый результат**: DELETE (200) -> GET (404)

## TC-16. Повторное удаление
**Метод**: Delete

**Описание**: Повторный вызов DELETE для уже удаленного id.

**Ожидаемый результат**: HTTP 404 (или 200, если реализована идемпотентность ""удаление = отсутствие""
# Нефункциональные тесты
## TC-17. Проверка времени ответа на запрос GET
**Метод**: GET

**Описание**: Проверка времени ответа на запрос GET /api/1/item/{id}

**Ожидаемый результат**: Время ответа (Response Time) < 200ms при нормальной нагрузке.
## TC-18. Попытка выполнить POST запрос без Content-Type
**Метод**: Post

**Описание**: Попытка выполнить POST запрос без Content-Type: application/json.

**Ожидаемый результат**: HTTP 415 Unsupported Media Type.

## TC-18. Сохранение с эмодзи в name
**Метод**: Post

**Описание**: Сохранение айтема с эмодзи в name.

**Ожидаемый результат**: Данные сохраняются и возвращаются без искажения кодировки.

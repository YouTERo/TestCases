# Отчет об обнаруженных дефектах

## BUG-01: Несоответствие регистра в поле Seller ID
- **Краткое описание:** В запросе POST ожидается `sellerID` (ID заглавными), а в ответе возвращается `sellerId` (d строчная).
- **Шаги:** 1. Выполнить POST `/api/1/item` с телом `{"sellerID": 123...}`.
  2. Проверить тело ответа.
- **Фактический результат:** В JSON-ответе ключ называется `sellerId`.
- **Ожидаемый результат:** Единообразие в именовании ключей согласно документации.
- **Серьёзность:** Влияет на удобство разработки и десериализацию

## BUG-02: Избыточный массив в ответе GET Item
- **Краткое описание:** При запросе конкретного ресурса по уникальному ID (`/api/1/item/{id}`) сервер возвращает массив объектов вместо одного объекта.
- **Шаги:** Выполнить GET `/api/1/item/{valid_uuid}`.
- **Фактический результат:** Ответ приходит в формате `[{...}]`.
- **Ожидаемый результат:** Ответ в формате `{...}`, так как ID уникален.
- **Серьёзность**: Усложняет логику обработки на клиенте

## BUG-03: Несоответствие структуры ответа POST `/api/1/item` документации
- **Краткое описание**: В Postman-коллекции указано, что успешный ответ содержит поле "id". На практике ID возвращается внутри строки в поле "status".
- **Шаги**: Выполнить POST запрос на `/api/1/item` с валидным телом.
- **Фактический результат**: `{"status": "объявление <uuid> создано"}`.
- **Ожидаемый результат**: `{"id": "<string>","sellerId": "<integer>","name": "<string>","price": "<integer>","statistics": {"likes": "<integer>","viewCount": "<integer>","contacts": "<integer>" }, "createdAt": "<string>"}`.
- **Серьёзность**: Блокирует автоматизацию и интеграцию без написания парсеров.

## BUG-04: Невозможность создания объявления с нулевыми значениями статистики
- **Краткое описание**: API возвращает ошибку 400 Bad Request при попытке передать значения 0 в полях статистики (likes, viewCount, contacts), считая их отсутствующими.
- **Шаги**: Отправить POST запрос на `/api/1/item` со следующим телом `{"sellerID": 123456, "name": "Test Item","price": 100, "statistics": {"likes": 0,"viewCount": 0,"contacts": 0}`
- **Фактический результат**: HTTP 400 Bad Request. В теле ответа сообщение об ошибке: `{"result":{"message":"поле likes обязательно","messages":{}},"status":"400"}`.
- **Ожидаемый результат**: `{"status": "объявление <uuid> создано"}`.
- **Серьёзность**: Невозможно создать новое объявление с честными (нулевыми) показателями. Пользователь или система вынуждены подставлять искусственные данные (минимум 1), что искажает бизнес-метрики и вводит покупателей в заблуждение.
